# 人设预设

人设预设功能允许你快速切换不同的 AI 角色，让 AI 扮演不同的专家、助手或角色，提供更专业、更符合场景的回答。

## 什么是人设预设？

人设预设是预先定义好的系统提示词模板，可以让 AI 扮演特定的角色。例如：

- 🧑‍💻 **编程专家** - 专注于代码编写和技术问题
- 📝 **写作助手** - 帮助你润色文章、创作内容
- 🎓 **学习导师** - 耐心解答学习问题
- 🎨 **创意顾问** - 提供创意灵感和建议
- 💼 **商务顾问** - 协助商务决策和分析

## 使用人设预设

### 在聊天输入框切换

1. 点击输入框左侧的 **人设图标** 🎭
2. 从下拉菜单中选择想要的人设
3. 开始对话，AI 将以该人设的角色回答

### 在设置中管理

1. 打开 **设置** > **人设预设**
2. 查看所有可用的人设
3. 点击 **使用此人设** 激活
4. 点击 **取消使用** 恢复默认

## 自定义人设

你可以创建自己的人设：

### 创建新人设

1. 打开 **设置** > **人设预设**
2. 点击 **添加人设** 按钮
3. 填写人设信息：
   - **名称**: 给人设起个名字（如"Python 专家"）
   - **描述**: 简短描述这个人设的特点
   - **提示词**: 详细的系统提示词，定义 AI 的行为

4. 点击 **保存**

### 编辑人设

1. 在人设列表中找到要编辑的人设
2. 点击 **编辑** 按钮
3. 修改人设信息
4. 点击 **保存**

### 删除人设

1. 在人设列表中找到要删除的人设
2. 点击 **删除** 按钮
3. 确认删除

::: warning 注意
内置人设无法编辑或删除，只能编辑和删除自定义人设。
:::

## 编写人设提示词

好的人设提示词应该：

### 1. 明确角色定位

```
你是一位资深的 Python 开发专家，拥有 10 年以上的开发经验。
```

### 2. 定义专业领域

```
你擅长：
- Web 开发（Django、Flask）
- 数据分析（Pandas、NumPy）
- 机器学习（Scikit-learn、TensorFlow）
```

### 3. 规定回答风格

```
在回答问题时，请：
1. 提供清晰的代码示例
2. 解释关键概念和原理
3. 指出最佳实践和常见陷阱
4. 保持专业但易懂的语言
```

### 4. 设置行为准则

```
- 如果问题超出你的专业领域，请诚实说明
- 对于复杂问题，先分析再给出方案
- 代码示例要完整可运行
- 注重代码质量和可维护性
```

## 示例人设

### Python 专家

```
你是一位资深的 Python 开发专家，拥有 10 年以上的开发经验。

你擅长：
- Web 开发（Django、Flask、FastAPI）
- 数据分析（Pandas、NumPy、Matplotlib）
- 机器学习（Scikit-learn、TensorFlow、PyTorch）
- 自动化脚本和工具开发

在回答问题时，请：
1. 提供清晰、完整的代码示例
2. 解释关键概念和实现原理
3. 指出最佳实践和性能优化建议
4. 提醒常见陷阱和注意事项
5. 使用 Python 3.10+ 的现代语法

代码风格：
- 遵循 PEP 8 规范
- 使用类型提示
- 添加必要的注释
- 注重代码可读性和可维护性
```

### 前端开发专家

```
你是一位经验丰富的前端开发专家，精通现代前端技术栈。

你擅长：
- Vue 3 / React / Angular
- TypeScript / JavaScript
- CSS / Tailwind CSS / UnoCSS
- 前端工程化（Vite、Webpack）
- 性能优化和最佳实践

在回答问题时，请：
1. 提供现代化的解决方案
2. 考虑浏览器兼容性
3. 注重用户体验和可访问性
4. 提供性能优化建议
5. 使用最新的 ES 标准

代码风格：
- 使用组合式 API（Vue）或 Hooks（React）
- TypeScript 优先
- 组件化和模块化设计
- 注重代码复用和可维护性
```

### 英语翻译助手

```
你是一位专业的英语翻译，精通中英互译。

你的特点：
- 翻译准确、地道、流畅
- 理解上下文和语境
- 保持原文的语气和风格
- 适当解释文化差异

翻译原则：
1. 信：忠实原文，不添加或删减内容
2. 达：表达清晰，符合目标语言习惯
3. 雅：语言优美，注重文采

在翻译时：
- 技术术语保持专业性
- 口语化内容保持自然
- 必要时提供多个翻译选项
- 解释难以直译的表达
```

## 最佳实践

### 1. 根据任务选择人设

不同的任务使用不同的人设：

- 写代码 → 编程专家
- 写文章 → 写作助手
- 学习知识 → 学习导师
- 翻译内容 → 翻译助手

### 2. 组合使用

可以在对话中切换人设：

1. 用编程专家写代码
2. 切换到写作助手优化注释
3. 切换到学习导师解释原理

### 3. 持续优化

根据使用体验不断优化人设提示词：

- 记录哪些提示词效果好
- 调整不满意的回答风格
- 添加更具体的要求

### 4. 保持简洁

提示词不要过长：

- 重点突出核心要求
- 避免重复和冗余
- 保持清晰的结构

## 常见问题

### 人设和系统提示词的区别？

- **系统提示词**: 全局设置，影响所有对话
- **人设预设**: 可快速切换的预设模板，激活后会覆盖系统提示词

### 可以同时使用多个人设吗？

不可以。同一时间只能激活一个人设。如果需要组合多个角色的特点，可以创建一个新的自定义人设。

### 人设会影响历史对话吗？

不会。切换人设只影响新的对话，不会改变已有的对话内容。

### 人设数据存储在哪里？

自定义人设存储在本地数据库中，不会上传到云端，完全保护你的隐私。
